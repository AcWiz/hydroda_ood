# 阶段 1：数据契约

## 目标

实现一个稳健的 DA increment emulation dataset wrapper。

## 必须实现的模块

```text
hydroda/data/dataset.py
```

## 任务

1. 实现 `HydroDADataset`。
2. 使用 xarray lazy-load NetCDF。
3. 根据 config 读取变量名和通道顺序。
4. 返回：
   - x
   - forecast
   - analysis
   - increment
   - obs_mask
   - loss_mask
   - region_mask
   - time_code
   - metadata
5. 支持 crop / padding。
6. 支持 patch sampling。
7. 保留 raw forecast / analysis 用于 increment 计算。

## 测试

创建：

```text
tests/test_dataset_contract.py
```

测试：

```text
shape 是否正确；
increment 是否等于 analysis - forecast；
mask shape 是否正确；
padding 区域 loss_mask 是否为 0；
除非明确允许，否则没有 NaN / Inf。
```

## 验收标准

- smoke script 能从美国 DA.nc 加载 8 个 samples。
- sample 包含所有 required keys。
- 在一个 batch 上计算 Forecast-only metric，Skill 必须等于 0。
