# HyperDA-OOD / HyperDA V4 完整研究计划

本文件为当前项目冻结版研究计划。旧版 HyRAO / sparse adaptation / simple-baseline 主表计划全部降级为历史方案或 internal sanity check。

## 0. 最终论文定位

这篇论文不写成“更好的土壤湿度预测模型”，而写成：

```text
HydroDA-OOD:
A leakage-controlled cross-continental benchmark for neural land data assimilation increment emulation.

HyperDA:
A hydroclimatic spatio-temporal prompt-conditioned hypernetwork for generating target-specific lightweight neural DA increment operators.
```

一句话中文版本：我们构建一个严格防泄漏的跨大陆 neural land DA increment emulation 协议，并提出 HyperDA，从 source operator episode bank 中学习 hydroclimatic spatio-temporal prompt 到轻量 DA operator 参数的映射，为未见目标区域生成 target-specific DA correction operator。

## 1. 核心科学问题

```text
Can hydroclimatic spatio-temporal prompts generate target-specific neural data assimilation increment operators that outperform shared conditional models and parameter-efficient few-cycle adaptation under cross-continental hydroclimatic shift?
```

我们真正回答：为什么不是 forecast-only，为什么不是 source-only，为什么不是 prompt-conditioned shared backbone，为什么不是 adapter / LoRA tuning，为什么需要 target-specific parameter generation。

## 2. 核心贡献

Contribution 1：HydroDA-OOD benchmark / protocol。严格区分 source train、target context/calibration 和 target query，并工程化防止 query-label leakage。

Contribution 2：HyperDA parameter-space transfer。不是把 prompt 拼到输入特征里，而是根据目标区域 prompt 生成 target-specific lightweight operator parameters。

Contribution 3：DA-specific deterministic basis-factorized generation。借鉴 parameter-space transfer 思想，但不照搬 diffusion 或 full-weight generation；科学 DA 场景优先稳定、可审计、轻量。

Contribution 4：可引用强 baseline 与可证伪分析。主表只比较 forecast-only、source-only、prompt-conditioned shared、adapter tuning、LoRA tuning、HyperDA-Zero、HyperDA-Calib、HyperDA-Refine。

## 3. 任务定义

任务是 neural land DA analysis-increment emulation：

```text
ΔSM_surface  = SM_surface_analysis  - SM_surface_forecast
ΔSM_rootzone = SM_rootzone_analysis - SM_rootzone_forecast
```

模型输出 `pred_increment_surface/rootzone`，再加回 forecast 得到 `pred_analysis_surface/rootzone`。

必须反复强调：

```text
We do not predict soil moisture from scratch. We emulate the correction operator induced by a physical / operational DA system.
```

## 4. 数据与区域

最终主实验使用 US/CN/AU，每个大陆 6 类 hydroclimatic regimes：R1 dryland、R2 semi-arid transition、R3 irrigated agriculture、R4 rainfed agriculture、R5 humid high vegetation、R6 mountain/cold terrain。

区域定义必须在训练前冻结，不能根据模型表现、analysis increments 或 target query labels 调整。

## 5. 主实验协议

Leave-one-continent-out：

```text
US + CN -> AU
US + AU -> CN
CN + AU -> US
```

时间协议：

```text
Source train: 2015-2020
Target context / calibration: 2021
Target query / evaluation: 2022-2025
```

K-cycle calibration：

```text
K ∈ {0, 4, 12}
```

K 是 DA cycles，不是 patches。

## 6. Leakage Guard

禁止 target query labels 用于 prompt construction、normalization、support-date selection、early stopping、model selection、threshold calibration、prompt feature tuning。

所有模块必须共享 `ProtocolConfig` 与 `LeakageGuard`：PromptBuilder、Normalizer、SupportSampler、Dataset、Evaluator、ExperimentRunner。

## 7. 方法：HyperDA-V4

```text
ζ_R = H_ψ(P_R)
ΔSM_hat_R = f_{θ0, ζ_R}(x_R)
```

组件：shared DA increment backbone、source operator episode bank、prompt encoder、basis-factorized hypernetwork parameter generator。

第一版不生成 full backbone weights，只生成 adapter、output-head residual、optional FiLM。

Basis factorization：

```text
ζ_R,l = ζ_0,l + Σ_m α_R,l,m B_l,m
```

## 8. 主版本

HyperDA-Zero：K=0，只用 2021 input-side spatial/temporal prompt。

HyperDA-Calib：K=4/12，把 calibration summaries 加入 prompt 直接生成 ζ_R。

HyperDA-Refine：从 HyperDA 生成的 ζ_R 初始化，在 K cycles 上只更新 ζ_R 10–50 步。

## 9. Baseline 体系

K=0：Forecast-only、Source-only backbone、Prompt-conditioned shared backbone、HyperDA-Zero。

K=4/12：Forecast-only、Source-only backbone、Adapter tuning、LoRA tuning、Prompt-conditioned shared + calibration prompt、HyperDA-Calib、HyperDA-Refine。

删除主表 baseline：source mean、target support mean、ridge、nearest-source、prompt-weighted、kNN interpolation、linear prompt-to-parameter。

## 10. 工程路线

Phase 0：DA.nc audit。

Phase 1：Region cropping，nc 与 pt 分文件夹，每个 region 单独保存。

Phase 2：HydroDADataset + ProtocolConfig + LeakageGuard。

Phase 3：Forecast-only + metrics/evaluation sanity。

Phase 4：Source-only ResUNet/ConvNeXt-UNet + prompt-conditioned shared backbone。

Phase 5：Source operator episode bank。

Phase 6：HyperDA-Zero。

Phase 7：K=4/K=12 calibration：adapter、LoRA、HyperDA-Calib、HyperDA-Refine。

Phase 8：US-only development report。

Phase 9：CN/AU 到位后扩展到 leave-one-continent-out。

## 11. 主指标

主文报告 Surface Skill、Rootzone Skill、Increment RMSE、Increment correlation、High-update Skill。

Forecast-relative skill：

```text
Skill = 1 - RMSE(SM_hat_analysis, SM_analysis) / RMSE(SM_forecast, SM_analysis)
```

## 12. 审稿风险

关键防御：任务不是 soil moisture prediction；K 是 cycles；query label zero leakage；Prompt-conditioned shared baseline 必须强；Adapter/LoRA 必须公平；source episodes 不是 independent domains，而是 structured local operator estimation problems。
