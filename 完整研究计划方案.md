# HydroDA-OOD：跨水文气候区 Neural Land Data Assimilation Operator 的少周期区域适配研究方案

> **目标定位**：面向 ICLR / NeurIPS / CVPR / ICML 等顶级会议，构建一个严格、可复现、可防御的 Scientific ML / Data Assimilation 交叉研究项目。  
> **核心主题**：在跨国家、跨水文气候区、跨观测误差分布的真实部署场景下，研究 neural land data assimilation operator 如何利用极少量目标区域 DA cycles 完成可靠区域适配。

---

## 0. 一句话版本

本文研究的问题不是“训练一个更好的土壤湿度预测模型”，而是：

> **当一个 neural land data assimilation operator 被部署到新的水文气候区域时，如何在只有 K=0/4/12/24 个目标区域历史 DA analysis cycles 的条件下，可靠、高效、稀疏地适配其 analysis-increment emulation 能力？**

我们将提出：

```text
HydroDA-OOD:
  A benchmark and protocol for few-cycle regional adaptation
  of neural land data assimilation increment emulators
  under hydroclimatic distribution shift.
```

---

## 1. 研究动机与问题定位

### 1.1 背景

Land data assimilation 系统通常将陆面模型 forecast、遥感观测、辐射传输相关变量和误差假设结合起来，生成 analysis soil moisture。近年来 neural operators / vision backbones / transformer-like architectures 被用于近似或加速 DA 过程，但这类模型在真实部署时会遇到显著的 **out-of-distribution generalization** 问题：

```text
1. 新国家 / 新大陆；
2. 新水文气候区；
3. 新植被覆盖与微波观测误差环境；
4. 新地形、雪、冻融、mask 分布；
5. 新农业管理与灌溉制度；
6. 目标区域只有少量历史 DA cycles 可用于校准。
```

因此，真实问题并不是在同分布随机划分下提升 RMSE，而是：

> **如何让一个 neural DA operator 在目标区域少量 DA cycles 下完成可控、稳定、可解释的 regional adaptation？**

---

### 1.2 本文不解决的问题

为了避免论文叙事被误解，本文明确不声称：

```text
We predict true soil moisture better than physical DA.
```

本文也不把任务定义为：

```text
generic soil moisture forecasting
```

而是严格定义为：

```text
neural land DA analysis-increment emulation
```

也就是说，我们学习的是 DA 系统从 forecast 到 analysis 的 correction / increment operator。

## 2. 任务定义

### 2.1 输入与目标

对于每一个 DA analysis cycle，合作方当前数据中的 DA 输入通道为：

```text
sm_surface_forecast
sm_rootzone_forecast
mwrtm_vegopacity
tb_h_obs
tb_v_obs
mask
```

目标通道为：

```text
sm_surface_analysis
sm_rootzone_analysis
```

我们不直接预测 analysis，而是预测 DA increment：

```text
ΔSM_surface  = SM_surface_analysis  - SM_surface_forecast
ΔSM_rootzone = SM_rootzone_analysis - SM_rootzone_forecast
```

模型输出：

```text
\hat{ΔSM_surface}, \hat{ΔSM_rootzone}
```

最终 estimated analysis 为：

```text
\hat{SM_surface_analysis}  = SM_surface_forecast  + \hat{ΔSM_surface}
\hat{SM_rootzone_analysis} = SM_rootzone_forecast + \hat{ΔSM_rootzone}
```

---

### 2.2 论文任务表述

推荐在论文中使用如下任务表述：

```text
Given a source-trained neural land DA increment emulator and an unseen target
hydroclimatic region, adapt the operator using only K target DA analysis dates
under strict query-label isolation.
```

中文解释：

> 给定一个在 source regions 上训练好的 neural land DA increment emulator，当模型部署到一个未见过的目标水文气候区域时，我们只允许使用 K 个目标区域历史 DA dates/cycles 进行校准，并要求在严格不接触 target query labels 的条件下评估其目标区域泛化能力。

## 4. 核心贡献

本文建议将核心贡献压缩为三个主贡献，避免主线发散。

---

### Contribution 1：HydroDA-OOD Benchmark

构建一个面向真实部署的跨区域 / 跨大陆 neural land DA increment emulation benchmark：

```text
Countries / continents:
  United States
  China
  Australia

Regime slots per country:
  R1 dryland / sparse vegetation
  R2 semi-arid grassland or agro-pastoral transition
  R3 irrigated / intensively managed agriculture
  R4 rainfed agriculture
  R5 humid high-vegetation
  R6 mountain / snow / cold-terrain stress
```

总计：

```text
3 countries × 6 regions = 18 hydroclimatic target/source regions
```

该 benchmark 的关键不是简单地扩大数据，而是提供：

```text
1. hydroclimatic OOD split；
2. country-level OOD split；
3. same-regime cross-continent transfer split；
4. few-cycle target adaptation protocol；
5. DA-oriented evaluation metrics。
```

---

### Contribution 2：K-date Few-Cycle Calibration Protocol

提出严格的 K-date support protocol：

```text
K ∈ {0, 4, 12, 24}
```

其中 K 表示：

```text
number of target DA analysis dates/cycles
```

而不是：

```text
number of patches
number of pixels
number of mini-batches
```

这是本文 benchmark 设计中最重要的防御点之一，因为空间 patch 数量很容易虚假放大 few-shot setting 的数据量。

---

### Contribution 3：HyRAO：Hydroclimate-conditioned Region-Adaptive Operator

提出区域条件化的 neural DA operator 少周期适配方法：

```text
HyRAO = Hydroclimate-conditioned Region-Adaptive Operator
```

HyRAO 由三部分组成：

```text
1. Input-only hydroclimate descriptor encoder；
2. Region-conditioned operator modulation；
3. Sparse K-date support adaptation。
```

核心思想是：

> 先用 forecast / observation / vegetation opacity / mask 等不含 analysis label 的输入变量构建目标区域 hydroclimate descriptor，再用极少量 target support DA cycles 对共享 DA operator 进行稀疏、稳定、区域特异的校准。

## 5. Benchmark 区域划分

### 5.1 区域划分原则

区域定义必须满足：

```text
1. 在训练前固定；
2. 不使用 analysis increment；
3. 不使用模型误差；
4. 不根据 query performance 调整；
5. 尽量依据地理、水文气候、植被、灌溉、地形和观测可用性先验；
6. 每个区域需要报告 valid pixel count、mask ratio、observation availability 等基本统计。
```

```text

vegetation opacity / land cover

snow / freeze-thaw relevance
observation mask and valid-pixel coverage
```

```text
9. descriptor supervision；

```

---

### 6.3 Normalization 规则

主实验建议使用：

```text
source-train-only normalization
```

即 normalization statistics 只能由 source train period 估计。

可选附录实验：

```text
source + unlabeled target support-year input normalization
```

但该设置必须明确标为 semi-supervised / input-only adaptation variant，不能与主实验混淆。

---

### 6.4 K=0 设置

K=0 是最容易被审稿人质疑的设置，需要明确两种版本：

#### 主实验：K=0-inductive

```text
只允许使用目标区域历史输入变量构建 descriptor；
不允许使用 target query labels；
不使用 target query period labels；
不做任何 support loss adaptation。
```

#### 附录：K=0-transductive

```text
允许使用 target query inputs 构建 input-only descriptor；
仍然禁止使用 query labels。
```

主论文建议以 K=0-inductive 为默认，以避免 transductive setting 争议。

## 7. K-date Support Selection

### 7.1 K 取值

```text
K = 0:
  不使用 target analysis labels；
  只允许 input-only hydroclimate descriptor。

K = 4:
  每个季节 / 水文季度选择 1 个 valid support date。

K = 12:
  每个月选择 1 个 valid support date。

K = 24:
  每个月选择 2 个 valid support dates；
  上半月和下半月各 1 个。
```

---

### 7.2 支持集采样规则

所有 support dates 必须满足：

```text
1. 只根据 calendar constraint 和 input-validity mask 选择；
2. 不根据 target analysis increment 选择；
3. 不根据模型误差选择；
4. 不根据 high-update magnitude 选择；
5. 不根据 query performance 选择；
6. 所有方法在同一个 region / K / seed 下使用完全相同的 support dates。
```

---

### 7.3 Support seeds

建议设置：

```text
minimum seeds: 5
preferred seeds: 10
```

每个结果报告：

```text
mean ± standard error across support seeds
```

对于主表格，至少报告：

```text
K = 0, 4, 12, 24
surface and rootzone
all target regions
```

## 9. 方法设计：HyRAO

### 9.1 总体结构

令共享 neural DA increment operator 为：

```text
f_θ(x, r) -> \hat{ΔSM}
```

其中：

```text
x:
  forecast, observations, vegetation opacity, mask, optional static covariates

r:
  region representation / hydroclimate descriptor
```

HyRAO 包含：

```text
1. shared DA backbone；
2. input-only descriptor encoder；
3. region-conditioned modulation module；
4. sparse support adaptation module。
```

---

### 9.2 Input-only Hydroclimate Descriptor Encoder

构造区域描述符：

```text
z_R = g_φ(S_R^input)
```

其中：

```text
S_R^input:
  target region 中的 input-only samples
  包括 forecast、TB_H、TB_V、vegetation opacity、mask 等；
  不包括 analysis；
  不包括 increment；
  不包括 query labels。
```

descriptor 可以由以下统计组成：

```text
1. forecast SM mean/std/quantiles；
2. TB_H / TB_V mean/std/missing ratio；
3. vegetation opacity distribution；
4. mask and valid observation ratio；
5. seasonal input statistics；
6. optional static covariates；
7. learned pooling features from input-only encoder。
```

---

### 9.3 Region-conditioned Modulation

将区域描述符用于调制 DA operator：

```text
h_l' = γ_l(z_R) · h_l + β_l(z_R)
```

或：

```text
h_l' = h_l + Adapter_l(h_l; z_R)
```

候选实现：

```text
1. conditional bias；
2. conditional normalization；
3. FiLM modulation；
4. region-conditioned adapter；
5. low-rank residual modulation；
6. region latent prompt token。
```

主方法建议从最稳定的版本开始：

```text
HyRAO-FiLM / HyRAO-Adapter
```

不建议一开始把 Hessian sparse tuning 作为唯一主方法。

---

### 9.4 Sparse K-date Support Adaptation

对于 K>0，使用 target support labels 进行少周期适配。

支持集损失：

```text
L_support =
  λ_surface · L( \hat{ΔSM_surface}, ΔSM_surface )
  +
  λ_rootzone · L( \hat{ΔSM_rootzone}, ΔSM_rootzone )
```

可选损失函数：

```text
1. masked MSE；
2. masked Huber loss；
3. high-increment weighted loss；
4. surface/rootzone balanced loss。
```

默认建议：

```text
masked Huber loss + surface/rootzone balanced weighting
```

稀疏适配策略：

```text
1. update only region latent；
2. update only adapter parameters；
3. update only modulation parameters；
4. update selected normalization affine parameters；
5. gradient Top-K module tuning；
6. Fisher / Hessian-informed sparse update as advanced variant。
```

主论文建议将：

```text
region latent + adapter / FiLM
```

作为主方法。

FISA / HISA / Hessian Top-K 建议作为高级 variant 或附录，只有在稳定提升后再升为主方法组成部分。

---

### 9.5 主方法命名建议

推荐名称：

```text
HyRAO: Hydroclimate-conditioned Region-Adaptive Operator
```

推荐论文中的方法描述：

```text
HyRAO adapts a shared neural DA increment operator to an unseen hydroclimatic
region by combining input-only regional descriptors, conditional operator
modulation, and sparse support-driven calibration under strict query-label
isolation.
```

## 11. Evaluation Metrics

### 11.1 主指标：Skill vs Forecast

主指标：

```text
Skill = 1 - RMSE(\hat{SM_analysis}, SM_analysis)
            / RMSE(SM_forecast, SM_analysis)
```

解释：

```text
Skill > 0:
  model improves over forecast

Skill = 0:
  model equals forecast

Skill < 0:
  model is worse than forecast
```

必须分别报告：

```text
surface
rootzone
```

---

### 11.2 Increment-space metrics

由于本文核心是 increment emulation，必须报告：

```text
Increment RMSE
Increment MAE
Increment correlation
Increment bias
Increment variance ratio
```

其中：

```text
ΔSM = SM_analysis - SM_forecast
\hat{ΔSM} = model output
```

---

### 11.3 Directional metrics

DA correction 的方向也很重要，因此报告：

```text
Sign accuracy with dead-zone
```

定义：

```text
只在 |ΔSM| > ε 的位置评估 sign；
ε 根据 source train increment distribution 固定；
不能根据 target query labels 调整。
```

---

### 11.4 High-update metrics

为评估模型是否能学到重要 DA correction，报告：

```text
Top-20% increment skill
High-update RMSE
High-update sign accuracy
```

High-update mask 定义：

```text
基于 target query ground-truth increment magnitude 的 top quantile；
只能在 prediction 后用于 evaluation；
不能用于 support selection、training、adaptation 或 model selection。
```

---

### 11.5 Few-cycle adaptation metrics

为了体现 K-date calibration 的价值，报告：

```text
Adaptation Gain
CE12
AUC-K
```

建议定义：

```text
Adaptation Gain(K) =
  Metric(method, K) - Metric(source-only, K=0)

CE12 =
  calibration efficiency at K=12

AUC-K =
  area under K curve over K ∈ {0,4,12,24}
```

对于 RMSE 类指标，应明确符号方向：

```text
Gain_RMSE(K) =
  RMSE(source-only) - RMSE(method, K)
```

对于 Skill 类指标：

```text
Gain_Skill(K) =
  Skill(method, K) - Skill(source-only)
```

## 15. 审稿人风险与防御策略

### Risk 1：这是不是只是 soil moisture prediction？

防御：

```text
不是。本文任务是 DA analysis-increment emulation。
我们学习的是 DA correction operator，而不是真实土壤湿度。
```

---

### Risk 2：few-shot 是否被空间 patch 数虚假放大？

防御：

```text
K 表示 DA dates/cycles，不表示 patch 数。
所有空间 patch 来自同一个 date 时仍然只算一个 support cycle。
```

---

### Risk 3：target query labels 是否泄漏？

防御：

```text
query labels 不用于 support selection、normalization、early stopping、
region selection、model selection、threshold calibration 或 high-update selection。
```

---

### Risk 4：区域是不是根据模型表现挑出来的？

防御：

```text
区域在训练前根据 hydroclimatic / land-cover / terrain / irrigation prior 固定。
不使用 analysis increment 或模型误差定义区域。
```

---

### Risk 5：simple baseline 是否足够强？

防御：

```text
将 forecast-only、source mean、target support mean、monthly support mean、
ridge calibration 作为主 baseline，而不是附录弱 baseline。
```

---

### Risk 6：方法提升是否只是因为参数更多？

防御：

```text
报告 trainable parameter count、adaptation compute、wall-clock time，
并比较 full fine-tuning、adapter、LoRA、bias-only、AdaBN 等参数高效 baseline。
```

---

### Risk 7：High-update 分析是否有选择偏差？

防御：

```text
High-update mask 仅用于 prediction 后 evaluation；
不能用于 support selection、training、adaptation 或 model selection。
```

---

### Risk 8：跨大陆数据到位前是否能形成论文？

防御：

```text
美国阶段作为 HydroDA-OOD-US development subset，
可用于验证 protocol、region OOD、K-date adaptation 和方法机制；
完整主论文仍应以 US/CN/AU 三大陆为目标。
```

## 18. 推荐摘要草稿

```text
Neural operators are increasingly used to emulate components of land data
assimilation systems, yet their deployment across unseen hydroclimatic regions
remains poorly studied. In practice, a target region may provide only a small
number of historical DA analysis cycles, making conventional fine-tuning
unstable and often ineffective. We introduce HydroDA-OOD, a benchmark and
protocol for few-cycle regional adaptation of neural land DA increment
emulators under hydroclimatic distribution shift. Rather than predicting soil
moisture directly, our task emulates DA analysis increments for surface and
root-zone soil moisture, with strict isolation of target query labels. We define
K-date calibration settings where K denotes the number of target DA cycles, not
the number of spatial patches, and evaluate models across dryland, semi-arid,
irrigated, rainfed, humid, and mountain/cold-terrain regimes. We further propose
HyRAO, a hydroclimate-conditioned region-adaptive operator that combines
input-only regional descriptors, conditional operator modulation, and sparse
support-driven adaptation. Experiments under leave-one-region and
leave-one-continent protocols evaluate analysis-space skill, increment-space
accuracy, directional correction, high-update performance, and calibration
efficiency. HydroDA-OOD provides a rigorous testbed for studying reliable
deployment of neural DA operators under realistic regional shifts.
```
