## 核心理念

代码库应该是可复现实验系统：

```text
configs 定义实验
scripts 运行确定性步骤
artifacts 保存决策
reports 解释有效性
tests 捕获泄漏和错误
```

## Config-first design

每个实验都应该可以从 config 复现：

```yaml
experiment_id: us_loro_forecast_v001
data:
  country: US
  nc_path: /fastersharefiles2/fenglonghan/dataset/SMAP/DA.nc
regions:
  split_file: artifacts/splits/us_loro/holdout_US_R1_K12_seed0.json
method:
  name: forecast
metrics:
  config: specs/metrics.yaml
```

## 测试优先级

```text
1. Dataset 返回正确 shape 和 finite tensors。
2. increment = analysis - forecast。
3. Forecast-only Skill = 0。
4. Split builder 不把 2023–2025 labels 放入 support。
5. K-date sampler 返回精确 K 个 unique dates。
6. Padding 区域 loss_mask=0。
7. Region masks 非空。
8. 同一 K/seed 的所有方法使用相同 support dates。
```
