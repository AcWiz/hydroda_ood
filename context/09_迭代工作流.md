# 09 Claude Code 迭代工作流

## 标准循环

每个阶段都按下面步骤执行：

```text
1. 阅读阶段任务卡。
2. 重述目标和约束。
3. 检查现有仓库结构。
4. 提出文件修改计划。
5. 实现最小可用版本。
6. 运行 tests 或 smoke checks。
7. 总结输出、失败和下一步。
8. 更新 decision log 和 open questions。
```

## 提示词来源

使用：

```text
prompts/Claude_Code_中文提示词.md
```

## 遇到不确定时

Claude Code 不应该编造数据假设。应该：

```text
1. 尽可能 inspect NetCDF；
2. 写 audit script；
3. 添加明确 TODO；
4. 用有帮助的错误信息 fail；
5. 请求缺失 metadata。
```

## 研究工程优先级

```text
correctness > no leakage > reproducibility > simple baselines > model novelty > speed
```

## 里程碑 gate

在下面条件满足前，不进入 neural modeling：

```text
NetCDF audit pass
dataset contract tests pass
US region masks exist
K-date splits exist
Forecast-only and mean baselines run
metrics sanity checks pass
```
