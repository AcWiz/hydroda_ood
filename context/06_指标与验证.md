# 06 指标与验证

## 主指标

Skill vs Forecast：

```text
Skill = 1 - RMSE(pred_analysis, analysis) / RMSE(forecast, analysis)
```

必须分别计算：

```text
surface
rootzone
```

## Increment metrics

对 predicted increments 计算：

```text
increment_rmse
increment_mae
increment_correlation
increment_sign_accuracy_deadzone
top20_increment_rmse
top20_increment_skill
```

## Sign accuracy dead-zone

不要在接近 0 的 increment 上计算 sign accuracy。

默认：

```text
epsilon = query valid nonzero |true_increment| 的 20th percentile
```

只在下面条件成立时计算 sign：

```text
|true_increment| > epsilon
```

## Top-20 high-update metrics

High-update mask 只用于 evaluation，并且只能在 prediction 后计算：

```text
top20_mask = |true_increment| >= valid query increments 的 80th percentile
```

严禁用于：

```text
support selection
adaptation
early stopping
model selection
```

## Few-cycle metrics

```text
AdaptationGain(K) = Skill(K) - Skill(0)
CE12 = Skill(K=12) / Skill(K=24)
AUC-K = area under Skill vs K curve
```

如果 Skill(K=24) 非正或非常不稳定，CE12 应标记为 unstable，而不是强行解释。

## Aggregation

默认主表：

```text
region-balanced mean
confidence interval over regions and support seeds
```

次要表：

```text
pixel-balanced mean
```

不能让大区域在主结果中主导平均值。

## 统计报告

每个方法报告：

```text
mean ± 95% CI over support seeds and regions
```

US-only leave-one-region-out 需要 per-region 表和 aggregate 表。

## Sanity checks

```text
Forecast-only Skill 必须等于 0。
Mean increment baseline 不能使用 target query labels。
Monthly support baseline 只能使用 support months。
如果 query month 没有 support estimate，必须按预定义规则 fallback。
```
