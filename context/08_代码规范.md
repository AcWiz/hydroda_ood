# 08 代码规范

## 风格

```text
Python 3.10+
public functions 使用 type hints
推荐 dataclasses 或 typed config
docstring 中写明 tensor shapes
避免 hidden global state
避免在代码中硬编码绝对路径
每个关键实验假设都要 logging
```

## Shape notation

统一使用：

```text
B: batch
C: channels
H: height
W: width
T: time
R: region
```

每个 tensor 都要标注 shape。

## 错误处理

以下情况必须 fail loudly：

```text
缺少 expected variables
time coverage 不足
region mask 为空
support dates 不足以满足 K
normalization stats 会使用 forbidden years
NaN / Inf 超过阈值
```

## 可复现性

每次 run 必须记录：

```text
git commit if available
config file
random seed
country / region / split IDs
K and support dates
normalization stats source
method hyperparameters
data path or hash
```

## 禁止事项

```text
不要静默裁剪 target query years。
不要使用 target query labels 做 normalization。
不要根据 performance 重定义 regions。
不要混淆 normalized units 和 raw units。
不要称 analysis 为 ground truth soil moisture。
不要在 dataset contract 测试前实现模型。
```
