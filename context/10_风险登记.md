# 10 风险登记

## Risk 1：把 analysis labels 误称为真实土壤湿度

应对：

```text
统一使用 DA analysis-increment emulation；
报告 toward reference DA analysis 的 Skill vs Forecast；
在论文 limitation 中说明 analysis label 的参考系统性质。
```

## Risk 2：区域选择被质疑 cherry-picking

应对：

```text
区域在看模型结果前固定；
使用 input-only descriptors 和外部科学区域知识；
保存 split JSON 和 metadata；
报告全部 6 个主区域。
```

## Risk 3：K-date 被质疑不是真 few-shot

应对：

```text
明确说 few-cycle calibration, not few-pixel learning；
报告 effective support budget；
所有方法使用相同 support dates。
```

## Risk 4：强简单 baseline 赢过复杂方法

应对：

```text
早期就跑 monthly increment 和 ridge；
如果它们很强，重新聚焦 benchmark value；
不要隐藏负结果。
```

## Risk 5：CN/AU schema 与 US 不同

应对：

```text
dataset adapter 必须 schema-driven；
构建 audit 和 variable mapping config；
避免 US hard-coding。
```

## Risk 6：mask 语义不清

应对：

```text
明确区分 obs_mask、region_mask、loss_mask；
给每个 mask 写 test；
在 region quality report 中可视化 mask。
```

## Risk 7：Sparse tuning 不稳定

应对：

```text
FISA/HISA 先作为 variant；
以 adapter / latent tuning 作为稳健核心；
报告 support seed 方差。
```
